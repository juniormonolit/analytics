# Смекалочная v1 — стартовый пакет для Cursor

Цель: собрать простое, быстрое и управляемое BI-приложение поверх Supabase, без конструктора DataLens и без лишней архитектурной сложности.

## Что делаем сейчас

Только раздел **Продажи**:

1. Отчет **По менеджерам**.
2. Отчет **По товарным группам**.

## Источник данных

Supabase, схема `sa`.

Основные таблицы:

- `sa.deals` — сделки. Базовая дата периода: `deals.created_at`.
- `sa.deal_events` — события по сделкам.
- `sa.teams` — отделы/команды.
- `sa.employees` — менеджеры.
- `sa.product_groups` — товарные группы.
- `sa.metrics` — каталог метрик.
- `sa.report_configs` — конфигурации отчетов, можно использовать как справочник системных отчетов.

## Главный принцип

Не строить универсальный BI-конструктор. Сначала делаем фиксированный runtime отчетов:

- фиксированные разделы;
- фиксированные отчеты;
- фиксированный набор базовых метрик из `sa.metrics`;
- фильтры сверху;
- таблица с периодом, сравнением, отклонением и drill down.

Все расширения потом.

## Стек

- Next.js App Router
- React
- TypeScript
- Tailwind CSS
- Supabase JS
- TanStack Query
- Zustand или URL state + local storage для UI-состояний

## UX-ориентир

Скриншоты пользователя:

- верхняя строка фильтров: период, отделы, прочие фильтры;
- выбор периода с пресетами справа;
- дерево отделов с чекбоксами и частичным выбором;
- модалка выбора показателей/наборов;
- группировки: Нет / Отдел / Итого;
- сохранение пользовательских настроек отчета.
