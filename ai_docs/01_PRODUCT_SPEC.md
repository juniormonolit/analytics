# Product Spec — BI v1

## Задача

Сделать приложение аналитики продаж, где руководитель быстро смотрит результат по менеджерам и товарным группам, сравнивает текущий период с предыдущим и проваливается в детализацию.

## Разделы сайдбара

Сайдбар скрываемый.

Основные пункты:

- Найм
- Маркетинг
- Продажи
- Реализация
- Настройки — внизу

В v1 активен только раздел **Продажи**. Остальные можно показать как disabled/coming soon.

## Структура раздела Продажи

Внутри раздела есть общие фильтры сверху. Они сохраняют значение на уровне раздела, а не конкретного отчета.

Отчеты внутри раздела отображаются:

1. как табы в рабочей области;
2. как вложенные ссылки в меню.

Отчеты:

- По менеджерам
- По товарным группам

## Фильтры раздела Продажи

Минимум:

1. Период.
2. Период сравнения.
3. Отделы.

Позже можно добавить:

- товарная группа;
- менеджер;
- воронка;
- тип сделки;
- стадия;
- филиал.

## Период по умолчанию

Дата фильтрации: `sa.deals.created_at`.

По умолчанию:

- текущий период: с 1-го числа текущего месяца по вчера;
- период сравнения: такой же по длительности хвост предыдущего месяца.

Пример: если сегодня 29.04.2026:

- текущий период: 01.04.2026–28.04.2026;
- сравнение: 03.03.2026–31.03.2026.

Важно: не сравнивать неполный месяц с полным прошлым месяцем. Сравниваем одинаковое количество дней.

## Пресеты периода

В date range picker справа:

- Сегодня
- Вчера
- Эта неделя
- Прошлая неделя
- Этот месяц
- Прошлый месяц

## Сравнение в таблице

Для каждой метрики показываем блок колонок:

- значение текущего периода;
- значение периода сравнения;
- Δ;
- Δ%.

Для компактности можно сделать режим отображения:

- компактный: `Текущий`, `Δ%`;
- полный: `Текущий`, `Сравнение`, `Δ`, `Δ%`.

В v1 делаем полный режим, но кодом заложить возможность компактного.

## Группировки

Переключатель группировки над таблицей:

- Нет
- Отдел
- Итого

Для отчета по товарным группам позже добавить:

- Филиал
- Отдел
- Менеджер

Группировка должна суммировать суммируемые метрики и корректно пересчитывать ratio/CR метрики из числителя/знаменателя, а не усреднять проценты.

## Drill down

При клике по строке открывается правая панель или модалка детализации.

Для отчета по менеджерам:

- первый уровень: менеджер;
- детализация: товарные группы внутри менеджера;
- следующий уровень: сделки.

Для отчета по товарным группам:

- первый уровень: товарная группа;
- детализация: менеджеры внутри товарной группы;
- следующий уровень: сделки.

## Сохранение настроек отчета

Сохранять персонально для пользователя:

- выбранные метрики;
- порядок метрик;
- скрытые/показанные колонки;
- ширины колонок;
- сортировка;
- режим группировки;
- возможно фильтры раздела.

В v1 можно хранить в localStorage, но сразу сделать abstraction `reportPreferencesRepository`, чтобы потом заменить на таблицу Supabase.
