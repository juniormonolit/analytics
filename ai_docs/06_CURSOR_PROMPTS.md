# Промты для Cursor

## Промт 1 — подготовь каркас приложения

Собери каркас BI-приложения v1 на Next.js App Router + TypeScript + Tailwind + Supabase.

Требования:

- русскоязычный UI;
- AppShell с сайдбаром и рабочей областью;
- скрываемый сайдбар;
- разделы: Найм, Маркетинг, Продажи, Реализация, Настройки внизу;
- активен только раздел Продажи;
- внутри Продажи табы: По менеджерам, По товарным группам;
- фильтр-бар сверху: период, отделы;
- пока данные можно замокать, но архитектуру сделать под Supabase.

Не делай универсальный BI-конструктор. Нам нужен фиксированный runtime отчетов.

## Промт 2 — подключи Supabase и report API

Подключи Supabase и сделай серверный endpoint `POST /api/reports/run`.

Источник данных: схема `sa`.

Период фильтруется по `sa.deals.created_at`.

Сделай два reportSlug:

- `by-managers`;
- `by-product-groups`.

API должен принимать period, comparisonPeriod, filters.teamIds, metricIds, grouping.

Сначала реализуй базовые агрегаты по `sa.deals`, затем подключим каталог `sa.metrics`.

## Промт 3 — период и сравнение

Реализуй DateRangePicker как на скрине:

- календарь;
- пресеты справа: Сегодня, Вчера, Эта неделя, Прошлая неделя, Этот месяц, Прошлый месяц;
- по умолчанию текущий период = с 1 числа текущего месяца по вчера;
- comparisonPeriod = такой же по длительности хвост предыдущего месяца.

Пример: сегодня 29.04.2026 значит current 01.04–28.04, comparison 03.03–31.03.

## Промт 4 — дерево отделов

Проверь реальную структуру `sa.teams`: есть ли `parent_id` или отдельная таблица иерархии.

Если есть иерархия — используй ее.
Если нет — сделай fallback на плоский список teams и подготовь код так, чтобы потом подключить `parent_id`.

Реализуй DepartmentTreeFilter:

- чекбоксы;
- раскрытие узлов;
- partial state;
- выбор родителя выбирает всех детей;
- в кнопку выводить `Отделы (N)`.

## Промт 5 — таблица отчета

Сделай ReportTable:

- sticky header;
- sticky first column;
- горизонтальный скролл;
- колонка dimension слева;
- для каждой метрики: current, previous, delta, deltaPercent;
- итоговая строка;
- сортировка;
- loading skeleton;
- empty state.

Форматирование: деньги, проценты, числа.

## Промт 6 — выбор показателей и сохранение настроек

Реализуй MetricPickerModal:

- вкладки Показатели и Наборы;
- поиск;
- теги;
- список метрик из `sa.metrics`;
- выбор/скрытие метрик;
- порядок метрик.

Сохраняй настройки персонально для пользователя через repository:

- сначала localStorage;
- интерфейс сделать так, чтобы потом заменить на Supabase table `sa.user_report_preferences`.

## Промт 7 — drill down

Сделай DrillDownPanel.

Для отчета По менеджерам:

- менеджер → товарные группы → сделки.

Для отчета По товарным группам:

- товарная группа → менеджеры → сделки.

Endpoint: `POST /api/reports/drilldown`.

Детализация сделок должна показывать: deal_id, deal_name, amount, created_at, stage_name, manager_id, team_id, product_group_name.
