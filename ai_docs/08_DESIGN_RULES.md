# Design Rules

## Язык

Весь пользовательский интерфейс на русском.

Исключения допустимы для общепринятых терминов:

- drill down;
- dashboard;
- API;
- Supabase;
- report;
- metric.

## Визуальный стиль

- Светлая тема по умолчанию.
- Чистые карточки.
- Мягкие границы.
- Табличный интерфейс должен быть плотным, но читаемым.
- Оранжевый можно использовать как акцент выбранных чекбоксов/активных элементов.

## Компоненты

Переиспользуемые компоненты:

- `AppShell`
- `Sidebar`
- `FilterBar`
- `DateRangePicker`
- `DepartmentTreeFilter`
- `ReportTabs`
- `ReportToolbar`
- `MetricPickerModal`
- `ReportTable`
- `DrillDownPanel`

## Состояния

У каждого крупного блока должны быть состояния:

- loading;
- error;
- empty;
- ready.

## Таблицы

- Не ломать ширину экрана.
- Горизонтальный скролл внутри таблицы.
- Первая колонка sticky.
- Header sticky.
- Итоговая строка закреплена снизу или визуально выделена.
